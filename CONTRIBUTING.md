# Contributing

Short orientation for humans. AI-agent rules live in
[CLAUDE.md](./CLAUDE.md); release tier rules live in
[VERSIONING.md](./VERSIONING.md).

## Dev environment

```sh
git clone https://github.com/k8rthik/cod-sync.git
cd cod-sync
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
pre-commit install   # sets up commit-time + push-time hooks
```

That's it — no native deps, no databases, no `.env`. The CLI is
self-contained.

## Running the tests

Default run is offline, ~250 tests, sub-second:

```sh
pytest -q
```

A small set of real-network integration tests against Moxfield and
Archidekt is gated behind an env var so they don't run by default:

```sh
COD_SYNC_RUN_NETWORK_TESTS=1 pytest -q tests/integration
```

They use a public Wizards-published Commander precon as the fixture;
if either source's API changes shape, the parity assertion between
them flags the regression. See
`tests/integration/test_sources_network.py` for the invariants.

## Lint, format, types

```sh
ruff check         # lint
ruff format        # apply formatter
mypy               # strict mode (see [tool.mypy] in pyproject.toml)
```

The pre-commit hook runs ruff on each commit and mypy on commit + push;
pytest runs on push. If any of them fails the hook blocks the commit /
push. Fix the underlying issue rather than bypassing with
`--no-verify` — see CLAUDE.md for the rationale.

## Project layout

- `cod_sync/cli/` — argument routing, prompts, sync/import/walk modes
  (all funnel through `_sync_deck` in `cli/sync.py`)
- `cod_sync/sources/` — per-source fetchers (Moxfield, Archidekt,
  plain text); each module follows the same shape
- `cod_sync/cod.py` — Cockatrice `.cod` file read/write
- `cod_sync/diff.py` — local vs. remote diff with DFC reconciliation
- `cod_sync/alt_name.py` — Scryfall reskin → canonical name cache
- `cod_sync/_seed_data.py` — generated artifact; do not hand-edit
  (regenerated by `scripts/refresh_seed.py`)
- `tests/` — mostly offline; `tests/integration/` is opt-in network

Module conventions: small, single-purpose files (200–400 lines
typical, 800 max). New deck sources should live under `cod_sync/sources/`
following the existing modules' shape.

## Public contract

Three surfaces have versioning weight (see
[VERSIONING.md](./VERSIONING.md)):

1. **CLI grammar** — positional args, flags, exit codes, stdout
2. **`.cod` file output** — what Cockatrice reads back
3. **Alt-name cache schema** — `~/.cache/cod-sync/alt_names.json`

Changes touching any of those need a tier classification. Internals
(module layout, function signatures, test helpers) don't.

## Commit conventions

Conventional Commits: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`,
`chore:`, `perf:`, `ci:`. Feature/fix work and the version bump are
separate commits in the same push. The bump touches exactly two files:
`pyproject.toml` and `cod_sync/__init__.py`, plus the historical table
at the bottom of `VERSIONING.md`.

## Releases

Cutting a release after a bump commit lands on `main`:

```sh
git tag v0.11.0
git push origin v0.11.0
```

The `release` workflow builds the wheel + sdist and attaches them to
a GitHub Release. cod-sync isn't published to PyPI today.
